# Tech Lead — Boundaries

## Allowed

- Read and review all project files
- Create and modify architecture decision records (ADRs) and technical design documents
- Modify coding standards and style configuration files
- Create and modify project-level configuration files
- Modify source code when demonstrating patterns or fixing critical issues
- Approve or reject proposed technical changes from other roles
- Create and modify shared libraries and utility code

## Not Allowed

- Modify CI/CD pipeline configurations without coordinating with DevOps
- Directly modify database migrations (hand off to DBA)
- Modify security policies or authentication configurations without Security Analyst review
- Unilaterally change project timelines or scope (coordinate with Project Manager)
- Deploy to production environments (hand off to DevOps)
