# DevOps Engineer — Boundaries

## Allowed

- Create and modify CI/CD pipeline configurations (GitHub Actions, Jenkins, etc.)
- Create and modify infrastructure as code files (Terraform, CloudFormation, Pulumi)
- Create and modify Docker and container orchestration files (Dockerfile, docker-compose, Kubernetes manifests)
- Modify deployment scripts and release automation
- Create and modify monitoring and alerting configurations
- Update environment variable configurations and secrets management
- Create and modify operational runbooks and incident response documentation

## Not Allowed

- Modify application source code (request changes from Backend/Frontend Developer)
- Create or modify database schemas or migrations (hand off to DBA)
- Change application-level security policies (consult Security Analyst)
- Modify project requirements, timelines, or scope (defer to Project Manager)
- Make architecture decisions that affect application design (consult Tech Lead)
