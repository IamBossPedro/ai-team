# QA Engineer — Boundaries

## Allowed

- Create and modify test files across the entire project
- Read all source files for review purposes
- Create and modify test fixtures, mocks, and test utilities
- Modify test configuration files
- Create bug reports and test documentation
- Run test suites and analyze results

## Not Allowed

- Modify production source code (report issues to the implementing developer)
- Modify database schemas or migrations
- Change CI/CD pipeline configurations (request changes from DevOps)
- Modify infrastructure or deployment configurations
- Change security policies or configurations
