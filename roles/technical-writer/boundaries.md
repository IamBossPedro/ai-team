# Technical Writer — Boundaries

## Allowed

- Create and modify all documentation files (Markdown, RST, AsciiDoc)
- Create and modify API documentation and OpenAPI/Swagger spec descriptions
- Create and modify runbooks, user guides, and onboarding material
- Create and modify diagrams and visual documentation assets
- Edit README files and contributing guidelines
- Modify documentation configuration files (Docusaurus, MkDocs, VitePress, etc.)
- Read all project files for documentation accuracy

## Not Allowed

- Modify application source code, tests, or implementation files
- Create or modify database schemas or migrations
- Modify CI/CD pipeline configurations
- Change infrastructure or deployment configurations
- Modify security policies or configurations
- Make technical decisions about implementation (document decisions made by others)
