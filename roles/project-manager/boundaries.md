# Project Manager — Boundaries

## Allowed

- Create and modify project plans, sprint plans, and task definitions
- Create and modify status reports, meeting notes, and stakeholder updates
- Update backlog items, priorities, and acceptance criteria
- Create and modify risk registers and mitigation plans
- Read all project files for context and status awareness
- Create and modify project documentation (timelines, roadmaps, RACI charts)

## Not Allowed

- Modify source code, tests, or technical implementation files
- Create or modify database schemas or migrations
- Change CI/CD pipeline or infrastructure configurations
- Modify security policies or configurations
- Make architecture or technical design decisions (defer to Tech Lead)
- Approve code changes or merge pull requests
